# /monitoring - 브랜드 모니터링 리포트

브랜드 및 경쟁사 온라인 반응을 자동으로 수집하고 분석하는 커맨드입니다.

## 사용법

```bash
/monitoring                           # 기본 모니터링 (일룸 브랜드)
/monitoring --keyword "일룸 책상"     # 특정 키워드 모니터링
/monitoring --period 7d               # 최근 7일 데이터
/monitoring --competitor              # 경쟁사 포함 비교 분석
```

## 프로세스

### 1단계: 검색 키워드 확인

사용자 CLAUDE.md에서 모니터링 키워드를 확인합니다:

```yaml
모니터링 키워드:
  브랜드:
    - 일룸, iloom
    - 일룸 책상, 일룸 침대, 일룸 소파
  경쟁사:
    - 한샘, 리바트, 에몬스
  트렌드:
    - 가구 트렌드, 홈오피스
```

키워드가 없으면 사용자에게 설정을 요청합니다.

### 2단계: 웹 검색 수행

WebSearch 도구를 사용하여 다음 소스를 검색합니다:

1. **네이버 블로그/카페** - 실사용 후기
2. **인스타그램** - 해시태그 기반 반응
3. **커뮤니티** (뽐뿌, 클리앙 등) - 솔직한 평가
4. **뉴스** - 공식 발표, 트렌드

검색 쿼리 예시:
- `일룸 리뷰 2026`
- `일룸 vs 한샘 비교`
- `가구 트렌드 2026`

### 3단계: 데이터 분석

수집된 데이터를 다음 기준으로 분석합니다:

**감정 분석**:
- 긍정: "좋아요", "만족", "추천", "예쁘다" 등
- 부정: "별로", "실망", "불만", "느리다" 등
- 중립: 정보 전달, 질문 등

**주제 분류**:
- 제품 품질 (디자인, 내구성, 조립)
- 서비스 (배송, AS, 상담)
- 가격 (가성비, 할인)
- 비교 (경쟁사 대비)

## 저장 위치

리포트는 다음 경로에 저장됩니다:
`10-projects/10-monitoring/{YYYY-MM-DD}_monitoring-report.md`

## 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--keyword` | 특정 키워드 지정 | `--keyword "일룸 침대"` |
| `--period` | 분석 기간 | `--period 30d` (7d, 14d, 30d) |
| `--competitor` | 경쟁사 비교 포함 | `--competitor` |
| `--source` | 특정 소스만 | `--source naver` |
| `--output` | 저장 경로 지정 | `--output ./reports/` |

## 주의사항

1. **API 제한**: 검색 결과는 최근 데이터 위주로 제한될 수 있습니다
2. **정확도**: 감정 분석은 참고용이며, 중요한 결정은 원문 확인 필요
3. **저작권**: 원문 인용 시 출처 명시 및 링크 제공

## 확장

### 커스텀 키워드 추가

CLAUDE.md의 모니터링 키워드 섹션에 추가:

```yaml
모니터링 키워드:
  브랜드:
    - 일룸
    - (새 키워드 추가)
```

### 자동화 연동

n8n 워크플로우와 연동하여 매주 자동 실행 가능:
- 트리거: 매주 월요일 09:00
- 액션: /monitoring --period 7d --competitor
- 알림: Slack 채널로 리포트 공유
